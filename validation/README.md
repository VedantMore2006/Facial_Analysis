# Validation Suite: MediaPipe vs OpenFace Feature Comparison

This directory contains validation scripts that compare MediaPipe feature extraction against OpenFace (ground truth).

---

## 📁 Files Overview

### Comparison Scripts

| Script | MediaPipe Feature | OpenFace Reference | Expected Correlation |
|--------|------------------|-------------------|---------------------|
| **compare_au12.py** | `au12_raw` (smile intensity) | `AU12_r` (lip corner elevation) | 0.6-0.8 (strong) |
| **compare_yaw.py** | `yaw_raw` (head rotation) | `pose_Ry` (head yaw angle) | 0.7+ (very strong) |
| **compare_ear.py** | `ear_raw` (eye aspect ratio) | `AU45_r` (blink/eye closure) | Negative correlation expected |
| **compare_expressivity.py** | `expressivity_raw` (landmark movement) | AU variance (facial animation) | 0.4-0.6 (moderate) |

### Utility Scripts

- **run_all_validations.py** — Master script that runs all comparisons sequentially
- **column_identification.py** — Helper to identify available columns in CSV files

---

## 🚀 Usage

### Run Individual Comparison

```bash
cd validation/
python compare_au12.py          # Compare smile detection
python compare_yaw.py            # Compare head pose
python compare_ear.py            # Compare blink detection
python compare_expressivity.py   # Compare facial animation
```

### Run All Validations at Once

```bash
cd validation/
python run_all_validations.py
```

## 📊 What Each Script Does

### 1. **Data Loading & Merging**
- Loads MediaPipe and OpenFace CSV files
- Merges on `frame_index` to align data
- Filters out invalid frames

### 2. **Normalization**
- Uses `StandardScaler` to z-score normalize both signals
- Ensures fair comparison despite scale differences

### 3. **Correlation Analysis**
- Computes Pearson correlation coefficient
- Provides p-value for statistical significance
- Interprets correlation strength

### 4. **Overlay Plot**
- Plots both normalized signals on same graph
- Shows correlation in title
- Saves to PNG file

### 5. **Smoothed Plot**
- Applies 10-frame rolling average
- Reduces landmark jitter noise
- Shows cleaner signal comparison

### 6. **Frame Alignment Check**
- Cross-correlation to detect temporal shift
- Shows lag in frames
- Warns if misalignment detected

## 🎯 Interpretation Guide

### Correlation Strength

| r value | Interpretation |
|---------|---------------|
| **0.8+** | Extremely strong — features track almost perfectly |
| **0.6-0.8** | Strong — features align well with minor differences |
| **0.4-0.6** | Moderate — features capture similar patterns |
| **0.2-0.4** | Weak — features show some relationship |
| **<0.2** | Very weak — features may measure different things |

### Feature-Specific Expectations

**AU12 (Smile):**
- Target: 0.6-0.8
- Both measure lip corner elevation
- Differences due to landmark selection

**Head Yaw:**
- Target: 0.7+
- Direct geometric measurement
- Should be highly correlated

**Eye Aspect Ratio (vs AU45):**
- Target: Strong NEGATIVE correlation (-0.6 to -0.8)
- EAR: High = open eyes, Low = closed eyes
- AU45: High = closed eyes, Low = open eyes
- Inverse relationship is correct!

**Expressivity:**
- Target: 0.4-0.6 (moderate)
- Different methods: movement magnitude vs AU diversity
- Moderate correlation expected

## 🐛 Troubleshooting

### Issue: `FileNotFoundError: mediapipe_raw_features.csv`

**Solution:**
```bash
# Make sure you're in the validation directory
cd validation/

# Check if CSV files exist
ls *.csv
```

### Issue: `ModuleNotFoundError: No module named 'pandas'`

**Solution:**
```bash
pip install pandas scikit-learn scipy matplotlib numpy
```

### Issue: Plots don't show

**Solution:**
- Plots are saved as PNG files even if display fails
- Check for `*_overlay.png` and `*_smoothed.png` files

### Issue: Very low correlation (<0.3)

**Possible causes:**
- Videos are from different sessions
- Frame alignment is off (check lag value)
- Feature computation has bugs
- One system failed to detect face properly
