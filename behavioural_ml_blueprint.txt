Behavioral Feature Dataset Generation & XGBoost Training Blueprint
1. Project Goal
The objective of this project is to build a machine learning model capable of estimating mental‑health risk signals
from behavioral facial data. The system will extract behavioral features from facial landmarks detected via MediaPipe
and use those features to train a supervised machine learning classifier (XGBoost).

The model will ultimately classify behavioral patterns into mental health categories such as:
• Healthy / Baseline
• Depression
• Anxiety
• Mania / Hypomania
• Burnout

The focus of this phase is not clinical diagnosis but behavioral signal detection. The model will learn statistical
patterns from engineered behavioral features derived from facial landmarks.
2. Core Feature System
The feature extraction pipeline produces 34 behavioral features from facial landmarks detected using MediaPipe.
These features represent four behavioral domains:

1. Facial Expressivity
2. Psychomotor Activity
3. Eye & Gaze Behavior
4. Temporal Response Behavior

Examples include:
• Smile intensity (AU12 mean amplitude)
• Brow tension (AU4 activation)
• Blink rate
• Gaze shift frequency
• Head velocity
• Facial emotional range
• Response latency
• Speech onset delay
• Pause duration

These features are derived from approximately 29 key MediaPipe facial landmarks covering:
• Brow region
• Eyelids
• Iris centers
• Mouth corners
• Lip edges
• Jawline
• Chin
• Nose bridge

Each extracted feature is normalized between 0–1 to allow stable machine‑learning training.
3. Dataset Unit Definition
The dataset will be constructed using 5‑second behavioral windows.

Each window becomes a single dataset row consisting of:
• 34 behavioral features
• 1 label (mental state class)

Example dataset row structure:

[au12_mean, au12_variance, blink_rate, head_velocity, facial_emotional_range, … reaction_time_instability_index] → label

Using window‑based samples dramatically increases dataset size and improves model learning stability.
4. Phase 1 – Baseline Video Data Collection
Record natural conversational sessions without attempting to simulate disorders.

Recommended recording plan:
• Total sessions: 20
• Duration per session: 3 minutes
• Total raw recording time: ~60 minutes

Recording content examples:
• Casual conversation
• Explaining a concept
• Reading text
• Answering questions
• Describing personal experiences

Goal:
Capture natural baseline facial behavior distributions.
5. Phase 2 – Feature Extraction Pipeline
Each recorded video will be processed using the existing behavioral feature extraction pipeline.

Processing pipeline:

Video Recording
      ↓
MediaPipe Face Mesh
      ↓
Facial Landmark Detection (29 landmarks)
      ↓
Behavioral Feature Computation
      ↓
5‑Second Window Aggregation
      ↓
Feature Dataset Generation

Expected output:

CSV dataset containing many rows like:

window_id | feature_1 | feature_2 | ... | feature_34
6. Phase 3 – Baseline Distribution Modeling
From the extracted dataset compute statistical distributions for each feature.

For each of the 34 features calculate:

• Mean
• Standard Deviation
• Minimum
• Maximum

Example:

AU12_mean_amplitude:
mean = 0.42
std = 0.11
min = 0.09
max = 0.68

These baseline distributions represent normal behavioral variability.
7. Phase 4 – Disorder Behavioral Modifiers
Synthetic mental‑state patterns will be generated by modifying baseline distributions.

Example behavioral effects:

Depression:
• Reduced smiling
• Reduced movement
• Increased response latency

Anxiety:
• Increased blink rate
• Increased brow tension
• Increased gaze shifts

Mania:
• Increased head motion
• Increased facial expressivity
• Faster responses

These effects are implemented using statistical multipliers applied to baseline feature means.
8. Phase 5 – Synthetic Dataset Generation
Synthetic samples will be generated from modified feature distributions.

Example generation formula:

value = random.normal(modified_mean, modified_std)

Dataset targets:

Healthy:     2000 samples
Depression:  2000 samples
Anxiety:     2000 samples
Mania:       2000 samples
Burnout:     2000 samples

Total dataset size:

10,000 rows
9. Phase 6 – Machine Learning Training
The synthetic dataset will be used to train an XGBoost classifier.

Dataset format:

X → 34 behavioral features
y → mental health label

Training configuration:

Train set: 80%
Test set: 20%

Example model configuration:

XGBClassifier
n_estimators = 500
max_depth = 6
learning_rate = 0.05
10. Phase 7 – Evaluation
Evaluate model performance using:

• Accuracy
• Precision
• Recall
• Confusion Matrix
• Feature Importance

Feature importance analysis will reveal which behavioral features contribute most strongly
to mental‑state prediction.
